  - Impact: Need one interactive prompt to capture ExplanationEngine runtime-status lines definitively.
- Decision: Add explicit no-active-profile error handling in chat send path.
  - Reason: Silent early return created a confusing "nothing happens" failure mode.
  - Impact: Users now receive actionable feedback and testers can immediately detect missing profile setup.
- Decision: Add on-demand profile reload in chat send path.
  - Reason: Chat ViewModel can outlive profile changes and hold stale `activeProfileId`.
  - Impact: Profile creation/switch on other screens no longer blocks chat send with stale error.
- Decision: Lower retrieval relevance threshold from 0.1 to 0.05.
  - Reason: Reduce false "no relevant content" misses on valid Grade 6 maths queries while still filtering noise.
  - Impact: More borderline-relevant chunks are considered, improving response availability.
- Decision: Render retrieval-miss messages inline in chat and suppress the global error banner for that case.
  - Reason: The previous UI stacked message-level and global errors for a single expected retrieval-miss path, causing confusion.
  - Impact: Cleaner UX with one clear assistant response instead of duplicate error signaling.
- Decision: Add Preferences "Apply and go back" action that returns to Profiles.
  - Reason: User requested explicit save/apply affordance and return flow.
  - Impact: Preferences screen now has a clear completion action aligned with user workflow.
- Decision: Fix preferences layout using `Modifier.weight(1f)` on content and `fillMaxWidth()` on LazyColumn instead of `fillMaxSize()`.
  - Reason: LazyColumn with `fillMaxSize()` consumed all vertical space, pushing the "Apply and go back" button off-screen and invisible.
