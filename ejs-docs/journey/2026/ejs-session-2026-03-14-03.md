# Decisions Made

- Decision: Gradle 8.11.1 with AGP 8.7.3
  - Reason: Latest stable Gradle 8.x and compatible AGP version as of early 2026
  - Impact: Supports Kotlin 2.1.0 compose compiler plugin and all target SDK features

- Decision: JVM target 17 (not 21)
  - Reason: JVM 17 is the current Android default and widely supported; JVM 21 is not yet standard for Android toolchains
  - Impact: Compatible with all CI environments and developer machines

- Decision: Dynamic colour on Android 12+ with fallback high-contrast scheme
  - Reason: Dynamic colour is a no-GMS feature (built into AOSP 12+); fallback scheme ensures WCAG AA on older devices and Huawei
  - Impact: Good colour accessibility without GMS dependency

- Decision: `android.nonTransitiveRClass=true`
  - Reason: Enforces module boundaries — each module only sees its own resources, preventing accidental cross-module resource leaks
  - Impact: Aligns with 4-layer architecture boundary rules from PRD §7.3

- Decision: `android:allowBackup="false"` in manifest
  - Reason: Learner data (profiles, preferences, chat history) must never leave the device (PRD §10, SP-01)
  - Impact: Prevents Android Auto Backup from sending data to Google Drive or any cloud service

- Decision: Configuration cache enabled (`org.gradle.configuration-cache=true`)
  - Reason: Significantly speeds up incremental builds on developer machines
  - Impact: May require minor compatibility fixes if plugins don't support it

# Key Learnings
Technical insights:
- Kotlin Compose compiler is now a Kotlin plugin (`kotlin.plugin.compose`) rather than a separate Compose compiler artifact — simplifies version management
- `installLocation="auto"` allows Android to move the app to adoptable storage (SD card), supporting the storage constraints in the PRD

# If Repeating This Work
Do this:
- Read PRD and architecture docs first to confirm SDK versions, constraints, and module structure
- Create root build files before module files (settings.gradle.kts defines the module graph)
- Use version catalogue from the start — never hardcode dependency versions in module build.gradle.kts

Avoid this:
- Don't add INTERNET permission even "just for debugging" — it violates the offline-first contract
- Don't add GMS dependencies even transitively (e.g., firebase-analytics comes via play-services)

Watch out for:
- Compose BOM version must match Kotlin version compatibility — verify the Compose-Kotlin compatibility map
- R8 rules need updating when native libraries (llama.cpp, ONNX Runtime) are added
- Feature module namespaces must be unique — use `com.ezansi.app.feature.{name}` pattern consistently
