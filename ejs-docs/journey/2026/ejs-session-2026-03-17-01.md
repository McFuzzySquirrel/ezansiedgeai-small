# Decisions Made

## D1: Manual DI over Hilt/Dagger/Koin
- **Decision:** Use Kotlin `lazy {}` singletons in AppContainer for dependency injection
- **Reason:** Zero reflection, zero code-gen, ~15 services total makes framework overkill. Compile-time safety.
- **Alternatives:** Hilt (GMS concerns, reflection), Koin (runtime overhead), Dagger (code-gen complexity)
- **Impact:** Simpler build, faster compilation, explicit dependency graph. Manual wiring required for new services.

## D2: AES-256-GCM with Android Keystore for profile encryption
- **Decision:** All profile data encrypted at rest using AES-256-GCM with keys from Android Keystore
- **Reason:** POPIA compliance, protection against physical device access, standard Android security practice
- **Impact:** Requires FakeAndroidKeyStore for unit tests. Slight overhead on profile reads/writes.

## D3: Sequential model loading for RAM management
- **Decision:** Embedding model unloads before LLM loads; never in memory simultaneously
- **Reason:** 4 GB device target = ~2 GB usable. Qwen2.5-1.5B needs 1,839 MB peak. Can't fit both.
- **Impact:** Slight latency increase during pipeline (unload + load), but stays within RAM budget.

## D4: Jinja2-style template engine (custom, no eval/reflection)
- **Decision:** Build pure recursive-descent parser supporting {{ }}, {% if %}, {% for %}, filters
- **Reason:** Security (no eval), offline (no deps), full control over grounding enforcement
- **Impact:** 784-line parser, but 98 unit tests provide confidence. Grounding instruction architecturally separated.

## D5: Theme.AppCompat for XML fallback theme
- **Decision:** Use AppCompat parent theme instead of MaterialComponents or Material3 in XML
- **Reason:** App uses Compose Material 3 — XML theme only needed for manifest/splash. AppCompat is already a dependency; adding google.material just for one XML style is wasteful.
- **Impact:** None on UI — Compose handles all actual theming.

# Key Learnings
1. **Sub-agents don't cross-validate dependencies.** Each agent produced correct code for its module but missed dependencies needed at the app level (material-icons-extended for library, activity-compose for topics). A build step is essential after agent orchestration.
2. **Gradle wrapper must be committed.** The scaffold had `gradle-wrapper.properties` but no `gradlew` script or jar — the wrapper generation step was missed during initial scaffolding.
3. **Phase 0 spike results are essential context.** Every agent needed the ADR/spike data to make correct decisions. The PRD + ADRs pattern works well for agent orchestration.
4. **Parallel agent execution works for independent modules.** Running learner-data-engineer and content-pack-engineer simultaneously saved time since they only shared core interfaces, not implementation details.

# If Repeating This Work
1. **Run `./gradlew assembleDebug` after each agent completes** — don't batch all compilation verification to the end. Catching errors earlier saves iteration.
2. **Include a "validate cross-module deps" step** in the orchestration plan. Each feature module should declare all its own dependencies, not rely on transitive deps from :app.
3. **Generate launcher icons during scaffold** — project-architect should create placeholder icons and Gradle wrapper as part of P0-101.
4. **Provide the version catalog to every agent** — agents made assumptions about available dependencies. Sharing `libs.versions.toml` content prevents phantom dependency references.

# Future Agent Guidance
1. **AppContainer.kt is the DI hub** — every new service must be wired here with `lazy {}`. No Hilt, no Koin.
2. **Feature modules need their own icon deps** — don't assume material-icons-extended is transitive. Add it explicitly if the module uses non-basic icons.
3. **Test with FakeAndroidKeyStore** — call `installFakeAndroidKeyStore()` in test setup and `FakeAndroidKeyStoreSpi.reset()` between tests.
4. **Template engine is in core:ai** — `TemplateEngine.kt` (784 lines) is a full Jinja2 renderer. Don't build another one.
5. **Mock implementations exist** — `MockEmbeddingModel` and `MockLlmEngine` allow the app to run without native libraries. Always maintain these for dev/test.
6. **Build command**: `cd apps/learner-mobile && ./gradlew assembleDebug` — gradlew is inside `apps/learner-mobile`, not the repo root.
