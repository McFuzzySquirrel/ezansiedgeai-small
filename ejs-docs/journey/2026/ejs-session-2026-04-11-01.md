# Decisions Made
- Decision: UNIFIED path — Gemma 4 for both generation and embedding
  - Reason: Single model load saves ~800 MB RAM, simpler architecture
  - Impact: All engines use shared GemmaModelProvider, content packs re-embedded to 768-dim

- Decision: Hash-based deterministic embedding as placeholder
  - Reason: MediaPipe doesn't expose embedding API yet; hash-based is reproducible and testable
  - Impact: Semantic search accuracy is random until real embedding available

- Decision: Retrieval-gated, not installation-gated for pack compatibility
  - Reason: Users should see their packs even if incompatible; clear error message guides update
  - Impact: PackVersionDetector checks at query time, not install time

# Key Learnings
Technical: Multi-agent parallel execution works well when file boundaries are clear (no conflicts in F4.2+F4.3 parallel)
Prompting: Providing full file contents in agent prompts produces better results than references
Tooling: Python venv isolation essential for pack building tools
