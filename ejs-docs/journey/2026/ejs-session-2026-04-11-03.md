# Decisions Made
- Decision: Remove temperature/topK/randomSeed from LlmInferenceOptions and rely on SDK defaults via implicit session
  - Reason: MediaPipe GenAI SDK 0.10.33 moved these params to LlmInferenceSessionOptions. The implicit session used by `generateResponse()` uses sensible defaults.
  - Impact: Model should now load successfully. Generation parameters use SDK defaults rather than explicit configuration. If fine-tuning is needed later, explicit LlmInferenceSession creation would be required.

- Decision: Use `setPreferredBackend(Backend.DEFAULT)` instead of `Backend.GPU`
  - Reason: Explicitly requesting GPU causes native crashes on devices without GPU accelerator support (vivo V2434). Using DEFAULT lets the SDK auto-detect the best available backend (falls back to CPU with XNNPack).
  - Impact: Model loads and inference works correctly on CPU. Second load is faster (7.4s vs 16.6s) thanks to XNNPack weight cache.

- Decision: Merge topic trees by path instead of naive addAll()
  - Reason: Multiple content packs can have overlapping root nodes (e.g. both have "term1"). LazyColumn requires unique keys.
  - Alternatives: Prefix keys with pack ID (breaks existing content references), deduplicate by dropping duplicates (loses content)
  - Impact: Topics screen loads reliably with any number of overlapping packs

- Decision: Make GemmaEmbeddingModel.isLoaded() always return true
  - Reason: Hash-based embedding doesn't need the Gemma model — it's always available. Gating on model load blocked search until first AI query.
  - Impact: Search works immediately on app launch, independent of model loading state

- Decision: Use startsWith() for bottom bar route matching
  - Reason: Route patterns include query params (e.g., "chat?question={question}") which don't match exact route string "chat"
  - Alternatives: Parse route to extract base path (more complex), register separate route for chat-with-question (would need 2 entries)
  - Impact: Chat tab correctly highlights when navigating via Ask AI

# Key Learnings
Technical insights: MediaPipe GenAI SDK 0.10.33 split model-level and session-level config. Model options handle resource allocation (path, max tokens, backend). Session options handle generation behavior (temperature, topK, seed).
Tooling insights: `javap` on extracted AAR classes.jar is the definitive way to check SDK API surface when docs are unclear.
